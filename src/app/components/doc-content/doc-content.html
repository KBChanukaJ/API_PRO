👥 GET /v1/users

GET

Retrieves a paginated list of all users associated with your project.

Rate Limit: 100 requests/minute
Auth Required: Yes
Scope: users:read

🔧 Query Parameters

Parameter Type Required Description
page integer Optional Page number for pagination (default: 1)
per_page integer Optional Number of items per page (default: 10, max: 100)
status string Optional Filter by user status: active, inactive, pending
search string Optional Search users by name or email

📋 Response Fields

users - Array of user objects
pagination - Pagination metadata

🔐 Authentication

Security

API PRO uses Bearer token authentication. All API requests must include a valid API token in the Authorization header.

🔑 Getting Your API Token

1

Login to Dashboard

Access your API PRO dashboard at dashboard.apipro.com

2

Navigate to API Tokens

Go to Settings → API Tokens in the sidebar

3

Generate New Token

Click "Generate New Token" and provide a descriptive name

4

Store Securely

Copy and store your token securely - it won't be shown again

📨 Authentication Header

HTTP Header 
Authorization: Bearer YOUR_API_TOKEN

🛡️ Security Best Practices

🚫
Never expose tokens in client-side code

Keep your API tokens on the server side only

🔄
Rotate tokens regularly

Generate new tokens every 90 days for enhanced security

🔒
Use HTTPS only

All API requests must use secure HTTPS connections

📦
Store in environment variables

Use environment variables or secure vaults for token storage

📋 Rate Limit Headers

X-RateLimit-Limit The maximum number of requests allowed per minute
X-RateLimit-Remaining The number of requests remaining in the current window
X-RateLimit-Reset The time when the rate limit window resets (Unix timestamp)
